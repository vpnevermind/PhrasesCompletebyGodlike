Description

We need to keep new phrases somewhere to have a stack to choose from. In this stage, you need to adjust your app using Room to store the phrases in a database.

Objectives

  1. Create an entity named Phrase for your phrases with the id and phrase as properties;

  2. Create the AppDatabase class based on a RoomDatabase;

  3. Don't use static methods to provide database instance to consumer code. Junit does not re-instantiate static variables and using the singleton pattern will not work properly with tests. Instead use an application class to instantiate the database and provide it to consumer code, like was shown on Room topic.

  4. Create a DAO object named PhraseDao and define methods as insert, delete, get, and getAll phrases from your database;

  5. The database name should be "phrases.db" and the table name should be "phrases";

  6. Update your previous code so that your RecyclerView shows the phrases from the database;

  7. When addButton is clicked show a dialog that will be used to insert more phrases in the database. Add an EditText with id "editText" to the dialog that will be used to insert the phrase to be added;

  8. Don't forget to update your RecyclerView: new phrases should pop up immediately!;

  9. Update your previous code so that the Delete TextView deletes the phrase from the screen and the database;

  10. Update your previous code so that your notifications take phrases from the database, not from the old list;

  11. Make sure you cannot set reminders with the empty database. If users try to set a reminder without phrases, show a Toast with an error message and set the reminderTv.text to No reminder set;

  12. If after a deletion the database gets empty no reminders should be set and existing ones should be canceled;
